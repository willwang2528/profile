# OpenClaw 公开边界

公开运行记录的目标是解释“有哪些 Agent、承担什么职责”，而不是复制内部配置。

## 可以公开

- Agent 的公开名称与职责
- 是否启用及大致运行状态
- 面向用户的能力范围
- 不含敏感参数的架构说明

## 禁止公开

- API Key、Secret、Token 与认证材料
- 用户、应用、群聊、会话等内部 ID
- 本地绝对路径、私有网络地址和设备标识
- 私有消息、提示词全文和未公开知识库内容
- 能间接恢复凭据或扩大权限的信息

## 发布原则

默认拒绝不必要的信息暴露。每次同步只发布完成公开说明所需的最少字段，并对生成结果进行人工可读检查。
