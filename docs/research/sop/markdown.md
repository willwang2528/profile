# 常用掌握的 Markdown 语法

> 已实践

## 1. 标题

```md
# 一级标题

## 二级标题

### 三级标题

#### 四级标题
```

在一篇笔记中，一般建议：

* `#`：笔记标题；
* `##`：主要章节；
* `###`：章节中的子问题。

例如：

```md
# GUI Agent

## 定义

## 核心模块

### 感知模块

### 规划模块

### 动作执行模块

## 相关论文
```

不要通过手动加粗代替标题：

```md
**核心模块**
```

因为真正的标题可以被目录、链接和大纲识别。

## 2. 加粗、斜体、高亮和删除线

```md
**加粗**

*斜体*

==高亮==

~~删除线~~
```

例如：

```md
GUI Agent 的核心问题是 **感知—决策—执行闭环**。

论文中最值得关注的是 ==任务成功率==，而不只是单步准确率。
```

`==高亮==` 和 `~~删除线~~` 属于 Obsidian 支持的 Markdown 扩展。([Obsidian][3])

---

## 3. 无序列表

```md
- 感知界面
- 识别控件
- 制定计划
- 执行动作
```

支持嵌套：

```md
- GUI Agent
  - 感知
    - OCR
    - 视觉编码
  - 决策
    - 任务规划
    - 动作预测
```

注意使用两个或四个空格进行缩进。

---

## 4. 有序列表

```md
1. 获取论文
2. 阅读摘要
3. 分析研究动机
4. 阅读方法
5. 检查实验
6. 总结贡献
```

---

## 5. 任务列表

```md
- [ ] 下载论文
- [ ] 阅读摘要
- [x] 阅读 Introduction
- [ ] 分析实验
- [ ] 判断是否复现
```

其中：

```md
- [ ] 未完成
- [x] 已完成
```

任务列表是 Obsidian 支持的 Markdown 语法之一。([Obsidian][3])

## 6. 引用

```md
> GUI Agent 的目标是让智能体通过图形用户界面完成真实任务。
```

多行引用：

```md
> 论文认为现有方法存在三个问题：
>
> 1. 缺乏长期规划；
> 2. 动作空间复杂；
> 3. 对界面变化不够鲁棒。
```

## 7. 代码

行内代码：

```md
模型通过 `click(x, y)` 执行点击操作。
```

代码块：

````md
```python
def click(x: int, y: int) -> None:
    print(f"Click at ({x}, {y})")
```
````

也可以指定其他语言：

````md
```bash
git clone repository_url
```

```json
{
  "action": "click",
  "x": 320,
  "y": 180
}
```
````

## 8. 分隔线

```md
---
```

例如：

```md
## 论文原文

原文内容……

---

## 我的理解

这部分主要解决……
```

但要注意：如果 `---` 出现在笔记最顶部，可能会被识别为属性区域。

---

## 9. 外部链接

```md
[显示文字](https://example.com)
```

例如：

```md
[论文项目主页](https://example.com)
```

外部链接使用标准 Markdown 的 `[文字](URL)` 格式。([Obsidian][5])

---

## 10. 图片

外部图片：

```md
![图片说明](https://example.com/image.png)
```

仓库内图片：

```md
![[agent-architecture.png]]
```

还可以控制图片宽度：

```md
![[agent-architecture.png|600]]
```

Obsidian 支持直接嵌入图片、音频、视频和 PDF 等文件。([Obsidian][6])

## 11. 数学公式

行内公式：

```md
损失函数为 $L = L_{\text{action}} + \lambda L_{\text{value}}$。
```

独立公式：

```md
$$
P(a_t \mid s_t, g)
=
\operatorname{softmax}(f_\theta(s_t,g))
$$
```

科研笔记中建议：

* 简单变量用行内公式；
* 完整推导使用独立公式；
* 每个公式后面补充自然语言解释。

例如：

```md
$$
a_t = \arg\max_a P(a \mid s_t, g)
$$

其中：

- $s_t$：当前界面状态；
- $g$：用户目标；
- $a_t$：智能体选择的动作。
```
